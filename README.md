# gitHub_palko12_vjezba_jedan
1. Što je Visual Studio Code i za što se najčešće koristi?
 
Visual Studio Code (VS Code) je lagani, besplatan i proširiv editor koda koji se koristi za pisanje, uređivanje i debugiranje različitih programskih jezika.
 
2. Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta?
 
Panel se zove Explorer.
 
3. Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe?
 
Ekstenzije se instaliraju preko Extensions panela. Služe za dodavanje novih funkcionalnosti kao što su podrška za jezike, teme, Git alati, formatiranje koda i mnoge druge opcije.
 
4. Kako otvoriti postojeći projekt ili mapu u VS Code-u?
 
Preko File → Open Folder, ili povlačenjem mape direktno u VS Code prozor.
 
5. Što se sve može raditi u Exploreru?
 
U Exploreru možeš: pregledavati datoteke i mape, stvarati nove datoteke/mape, brisati, preimenovati, premještati i otvarati ih.

6. Kako možeš brzo pronaći određenu datoteku u projektu?
 
Pritiskom Ctrl + P i upisivanjem imena datoteke.
 
7. Kako možeš istovremeno gledati dvije različite datoteke u editoru?
 
Koristiš Split Editor ili povučeš tab u desnu stranu editor prozora.
 
8. Kako pokrećeš automatsko formatiranje koda u VS Code-u?
 
Pritiskom Shift + Alt + F ili opcijom Format Document u izborniku.
 

9. Što je version control i zašto se koristi?
 
Version control je sustav za praćenje promjena u projektu.
Koristi se za sigurnu povijest promjena, rad u timu, povratak na starije verzije i bolju organizaciju koda.
 
10. Kako se u VS Code-u otvara panel za Git (Source Control)?
 
Klikom na ikonu Source Control (ikona grane) ili tipkama Ctrl + Shift + G.
 
11. Što znače commit, push, pull?
 
Commit – spremanje promjena lokalno.
 
Push – slanje promjena na GitHub.
 
Pull – preuzimanje promjena s GitHuba.
 
12. Čemu služi commit poruka i kako treba izgledati?
 
Commit poruka opisuje što je promijenjeno. Treba biti kratka, jasna i opisna (npr. “Dodana nova funkcija za login”).

13. Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci?
 
U Source Control panelu klikom na datoteku, gdje se prikazuje diff (razlika).

14. Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub?
 
1. git add .
2. git commit -m "Opis promjene"
3. git push

15. Kako se vraćaš na stariju verziju datoteke ako si pogriješio?
 
Podešavanjem preko povijesti:
ekstenzija “Git: View File History”,
naredba git checkout <commit> -- datoteka,
ili vraćanje cijelog commita pomoću git revert.

16. Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene?
 
Pomoću git status ili kroz Source Control panel.
